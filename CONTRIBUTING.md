# Contributing to the Vancouver Web Summit 2026 — GitHub Copilot SDK Hackathon

Thank you for participating! 🎉 This guide walks you through how to build and submit your hackathon entry.

## 🏁 How to Get Started

1. **Fork or create** your own GitHub repository for your project
2. **Build** your solution using the [GitHub Copilot SDK](https://github.com/github/copilot-sdk)
3. **Follow** the recommended repo structure (see below)
4. **Submit** via the [Microsoft Form](https://forms.office.com/r/DA6t38w4D5) before **April 26, 2026 — PST Midnight**

## 📂 Recommended Repo Structure

```
your-project/
├── /src or /app          # Working code
├── /docs
│   └── README.md         # Problem → Solution, prereqs, setup, deployment,
│                         # architecture diagram, RAI notes
├── /presentations
│   └── YourSolutionName.pptx   # Or a link to your deck
├── AGENTS.md             # Custom agent instructions
└── mcp.json              # MCP server configuration (if applicable)
```

## ✅ Submission Checklist

Before submitting, make sure you have:

- [ ] **Short summary** (max 150 words)
- [ ] **Demo video** (max 3 minutes) — uploaded and link ready to share
- [ ] **Working solution** in a public GitHub repo
- [ ] **README** with: problem statement, solution, prerequisites, setup instructions, architecture diagram, and RAI notes
- [ ] **1–2 slide deck** with business value proposition and architecture diagram
- [ ] **AGENTS.md** with custom agent instructions (if applicable)
- [ ] **mcp.json** with MCP server configuration (if applicable)

## 📹 Demo Video Tips

- Keep it under **3 minutes**
- Show the **working solution** — not just slides
- Tell a story: **problem → agent in action → outcome**
- Upload to YouTube, Loom, OneDrive, or any accessible platform
- Share the **video link** in the submission form
